# CLAUDE.md - WBS/EVM エコシステム憲法

## 基本理念: 「検証と修復 (Validation & Recovery)」
Excel への直接編集を許容しつつ、その整合性をプログラムによって担保する。「Excel は UI (出力) であり、Python スクリプトが真実のソース（SSOT）である。」また、単なる作業代行ではなく、プロジェクトマネジメントの理論的背景に基づいた洞察と教育的配慮を提供する。

## 1. 整合性チェックの原則
Excel ファイルの変更が行われた際、または Claude が作業を開始する際は、必ず整合性チェックを行う。
- **対象**: プロジェクト内のすべての `.xlsx` ファイル。
- **方法**: `python scripts/check_wbs.sh` を実行。
- **目的**: 破損したセル、不整合なデータ、壊れた参照の早期発見。

## 2. 異常検知および修復フロー
整合性チェックでエラーが検出された場合、以下のプロセスに従う。
1. **原因特定**: エラーログから、どのシートのどのセルに問題があるかを特定する。
2. **影響範囲の調査**: そのエラーが他のデータや計算にどのように波及しているかを調査する。
3. **修復計画の策定**: スクリプトによる自動修復が可能か検討し、計画を立てる。
4. **修復の実行と再検証**: 修復後、再度整合性チェックを行い、正常であることを確認する。

## 3. 理論的裏付けと教育的配慮 (Theoretical Rigor & Mentoring)
分析や提案を行う際は、単なる数値の提示に留まらず、以下の要素を付加すること。
- **PM理論に基づく解説**: 整合性エラーや指標の変化に対し、PMBOK等のプロジェクト管理理論（リソース競合、論理的順序など）を用いた背景説明を行う。
- **教育的フィードバック**: ユーザーの管理能力向上を支援するため、計画の堅牢性などを評価・賞賛する。

## 4. データ変更のルール
データの変更を行う際は、以下の原則を遵守する。
- **スクリプト優先**: 可能な限り、Excel の直接編集ではなく、Python スクリプト（`scripts/*.py`）を介してデータを更新する。
- **履歴の保持**: 重要なデータの変更は、変更ログを残すか、Git 等のバージョン管理システムで追跡可能にする。
- **整合性の維持**: 変更後には必ず整合性チェックを実行し、成功を確認する。

## 5. 使用する主要ツール
- **Python スクリプト**: `scripts/` ディレクトリ配下のツール群。
- **整合性チェックツール**: `python scripts/check_wbs.sh`
- **データソース**: `data/` ディレクトリ配下の CSV/JSON ファイル。
