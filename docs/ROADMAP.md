# WBS/EVM Ecosystem Roadmap

本プロジェクトは、Gemini CLI と Excel を高度に連携させ、プロジェクトマネジメントを「科学」に変えるための長期的な旅路を定義する。

## フェーズ 1: 基盤構築 (Done)
- [x] プロジェクト憲法 (GEMINI.md) の策定
- [x] Python 仮想環境と TDD 開発基盤の構築
- [x] スクリプト駆動型エクセルテンプレート (master_template.xlsx) の生成
- [x] GitHub によるバージョン管理の開始

## フェーズ 2: 運用の自動化と「スキル化」
エクセルを「手で更新する」手間を最小限にし、AIが能動的に管理する環境を構築する。
- **データ注入スクリプト**: 外部データ（CSV, 他のWBS等）から実績値を読み取り、テンプレートへ安全に流し込む。
- **自己修復バリデーター**: 運用中に壊された数式や構造を検知し、スクリプトに基づいて自動復旧する。
- **Geminiスキルの開発**:
    - `update-wbs`: 対話形式で進捗を報告すると、裏側でエクセルを更新する。
    - `check-integrity`: データの矛盾（開始日 > 終了日など）を即座にスキャンして報告する。

## フェーズ 3: インテリジェンス & 教育
数字を「分析」し、プロジェクトの「未来」を予測する知能を搭載する。
- **推移分析レポート**: CPI/SPI の履歴を記録し、週次のトレンドレポートを自動生成。
- **将来予測 (EAC/ETC)**: 現在の効率に基づき、完遂時期と最終コストを科学的に予測。
- **教育メンターモード**: 異常値が検出された際、「なぜこれが問題なのか」を PMBOK 理論に基づいて解説し、初学者の学びを支援する。

## フェーズ 4: 戦略的ポートフォリオ管理
複数の WBS を横断的に管理し、リソースの最適化を図る。
- **横断ダッシュボード**: 複数プロジェクトの EVM 指標を一括表示。
- **リソース・レベリング**: 特定のメンバーへの負荷集中を検知し、平準化案を提示。

---
## 運用時のゴール (North Star)
ユーザーは Gemini と対話するだけで、エクセルという「確かな証拠」を伴った精緻なプロジェクト管理を、苦労なく、かつ教育的な気づきを得ながら継続できる状態。
