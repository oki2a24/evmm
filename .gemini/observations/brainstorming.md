# 知見: WBS/EVM プロジェクトにおける計画策定のエッセンス

本プロジェクトにおいて `writing-plans` および `brainstorming` を使用する際、以下の 4 つの核心的エッセンスを自律的に遵守せよ。これらはユーザーの指摘を待たず、設計の前提（OS）として適用しなければならない。

## 1. Script-as-Master (スクリプト駆動)
- Excel ファイルは Snapshot に過ぎない。
- 数式、構造、データ処理の SSOT (Single Source of Truth) は常に Python スクリプトに置く。
- Excel の直接編集に依存する設計を避け、常に「スクリプトによる再現性」を確保せよ。

## 2. PMBOK 理論への厳格な準拠
- 指標名（BAC, EAC, CPI, SPI, VAC, TCPI）および計算公式は、PMBOK 標準をそのまま適用せよ。
- 独自の解釈を混ぜず、専門家（人間）が即座に理解できる「標準語」でドキュメントを記述せよ。

## 3. AI 用 JSON サマリーの構造化
- 分析エンジンの出力は、人間用のテキストだけでなく、機械可読な JSON を主とせよ。
- 複雑な計算を LLM に行わせず、スクリプトが算出した JSON 数値を基に AI が推論を行う「数学的ガードレール」を設計せよ。

## 4. 物理的証拠による完了定義 (DoD)
- 「完了」は、テストのパス、ビルドの成功、または検証スクリプトの出力によってのみ証明せよ。
- 計画には必ず「期待される物理的出力（コマンド結果等）」を明記し、事実に基づかない報告を排除せよ。
