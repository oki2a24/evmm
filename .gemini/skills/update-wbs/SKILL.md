# update-wbs

## 概要
ユーザーの自然言語による進捗報告を解析し、Excel WBSを整合性を保ちながら自動更新するためのスキルです。

## いつ使用するか
- ユーザーが「機能Xが完了した」「機能Yを3hでやった」などの進捗報告をした時。
- WBSの進捗率、実績工数、実績日付を更新する必要がある時。
- ユーザーが明示的に `/update-wbs` と指示した時。

## コア規律
**「推測で更新せず、常に物理的な整合性を証明せよ。」**

1. **ステートフルなフェーズ特定**:
   - 現在のWBSの進捗率を読み取り、次に更新すべきフェーズ（作成 -> レビュー -> 修正）を自動で特定せよ。
2. **工数実績の明示的確認**:
   - ユーザーから工数の指定がない場合、必ず「予定通りの◯hで更新してよいか」を確認せよ。勝手に更新してはならない。
3. **整合性チェックの強制 (Hard Gate)**:
   - Excelへの書き込み前に、必ず `scripts/check_wbs_integrity.py` を実行せよ。
   - 整合性エラー（日付逆転、過負荷など）がある場合は、保存を中断し、PM理論に基づいて理由を解説せよ。
4. **承認フローの遵守**:
   - 全てのパラメータが揃った後、「以下の内容で更新しますか？」と最終確認（はい/いいえ）を求めよ。

## 実装
バックエンドロジックは `scripts/update_wbs.py` に集約されています。
スキルはこのスクリプトを以下の形式で呼び出します。

```bash
.venv/bin/python3 scripts/update_wbs.py --id [機能ID] --phase [フェーズ] --effort [工数] --date [日付]
```

## よくある間違い
- **工数の無断入力**: 予定値があるからといって、確認なしに実績値を埋めてしまう。
- **整合性チェックのスキップ**: 「小さな更新だから」という理由で検証を怠る。
- **フェーズの誤認**: 前のフェーズが完了していないのに、次のフェーズを更新してしまう。
