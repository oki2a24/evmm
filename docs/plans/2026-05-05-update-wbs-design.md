# 対話型WBS更新機能 (update-wbs) 設計ドキュメント

## 1. 目的
ユーザーの自然言語による進捗報告を解析し、Excel WBSを整合性を保ちながら自動更新する。これにより、PMの入力負荷を軽減しつつ、正確なEVM分析結果をリアルタイムで得られる状態にする。

## 2. コア仕様

### 2.1 ステートフルなフェーズ判別
ユーザーが機能ID（例：「F1」）の進捗を報告した際、現在のWBSの進捗率を読み取り、次に更新すべきフェーズを自動的に特定する。
- 「作成」フェーズが未完了（<100%）なら → 「作成」を更新。
- 「作成」完了かつ「レビュー実施」未完了なら → 「レビュー実施」を更新。
- 以降、同様に「レビュー後修正」へと遷移する。

### 2.2 工数実績の入力と補完
- ユーザーに「工数（実績）」の入力を明示的に促す。
- 入力があった場合はその値を採用。
- 入力がない場合は「工数予定」の値をデフォルト値として使用し、ユーザーにその旨を通知する。

### 2.3 日付の自動設定
- 終了日実績：指定がなければ実行時の「今日の日付」を設定。
- 開始日実績：そのフェーズの最初の更新であれば「今日の日付」を設定（既存データがあれば維持）。

### 2.4 物理的整合性の強制 (Hard Gate)
Excelへの書き込み前に、必ず `scripts/check_wbs_integrity.py` のロジックを実行する。
- 整合性エラー（例：日付の逆転、リソース過負荷）を検知した場合は**保存を中断**し、ユーザーにPM理論に基づいた解説と共に修正を促す。

## 3. ユーザーインターフェース (トリガーとタイミング)

### 3.1 任意のタイミングでの実行
- **即時実行**: 「F1の作成を3hで完了して」など、具体的な指示があった場合は即座に更新・検証・報告を行う。
- **バッファリングと一括実行**: ユーザーが「昨日はF1とF2を終わらせた」と複数の報告をした場合、全てのパラメータが揃ったことを確認してから、またはユーザーの「反映して」という指示をトリガーに一括でExcelを更新する。
- **AIからの能動的確認**: パラメータが不足している（例：工数が不明）場合、AIは「工数は何時間でしたか？ 予定通りの2hで更新してもよろしいですか？」と確認し、ユーザーの承認（OK/Go）を得てから実行する。

## 4. システム構成

### 4.1 構成要素
- `scripts/update_wbs.py`: 更新ロジックの本体。
- `scripts/check_wbs_integrity.py`: 既存の検証ロジック（インポートして再利用）。
- `openpyxl`: Excelへのセル単位での書き込み（書式維持のため pandas ではなく openpyxl を使用）。

### 4.2 データフロー
1. **Input**: ユーザー発言（機能ID、工数、進捗など）
2. **Analysis**: ステートフルなフェーズ特定 + パラメータ抽出
3. **Pre-check**: 更新後のデータで整合性チェックを実行
4. **Action**: (Success) Excel保存 / (Failure) ユーザーへエラー解説

## 5. 成功基準
- 自然言語（例：「F1の作成が終わりました。3hかかりました」）でExcelの正しいセルが更新されること。
- 更新直後に `check_wbs.sh` を実行して「正常」と判定されること。
- 矛盾した入力（例：未来の日付での完了報告）が阻止されること。
