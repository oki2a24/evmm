# 自己修復バリデーター設計書 (案C: 一貫性駆動)

## 1. 背景と目的
「Script-as-Master」の思想に基づき、Excelの数式が破損（誤消去、直打ち上書き、参照ミス）した際、正解のロジックに基づいて外科的に修復する機能を提供する。
人間がExcelを直接編集する柔軟性を損なわず、かつ「行ごとの計算ロジックの一貫性」を物理的なガードレールとして維持する。

## 2. コア・アルゴリズム: 多数決パターンスキャン
1. **スキャン対象の特定**:
    - `WBS_EVM` シートの各データ行（3行目以降）を対象とする。
    - 数式（`=`で始まる）が含まれる列を自動的に検出、または特定の列（PV, EV, AC等）をターゲットとする。
2. **数式のテンプレート化**:
    - 各セルの数式から行番号を抽象化する。
    - 例: `A3`セルの `=B3+C3` -> `={column_B}{row}+{column_C}{row}`
3. **SSOT (正解) の決定 (多数決)**:
    - 各列において、最も多く出現するテンプレートを「正解」とみなす。
    - ユーザーが1行を修正し、それを多数の行に反映（オートフィル）させた場合、それが「新しい正解」として自動学習される。
4. **検知と修復**:
    - 正解パターンと異なるセル、または数式であるべき列に数値が直書きされているセルを特定し、正解テンプレートを再注入する。

## 3. 実装詳細
- **言語**: Python 3.12 (openpyxl)
- **新クラス**: `FormulaIntegrityManager` (scripts/utils.py または独立したファイル)
- **統合先**: `scripts/check_wbs_integrity.py` に組み込み。
- **CLI**: `check_wbs.sh` に `--fix` フラグを追加し、このフラグがある場合のみ修復を実行。

## 4. 発見性と自律性の担保 (Discoverability & Autonomy)
AIが迷わずに、即座に正しいアクションを取れる状態にするため、以下の物理的な仕組みを導入する。
- **専用スキルの提供**: `.gemini/skills/self-healing-validator/SKILL.md` を作成し、修復ロジックの SSOT とする。AIは「壊れた」という報告や不整合の検知を受けた際、即座にこのスキルを起動して手順を確認できる。
- **物理的ヒントの埋め込み**: `check_wbs.sh` が数式の不整合を検知した際、出力の末尾に「`--fix` フラグで修復可能」である旨のガイドを動的に表示する。これにより、AIは思考プロセスをショートカットして解決策を実行できる。

## 5. テスト戦略 (TDD)
- **Extraction Test**: 様々な数式（相対参照、絶対参照、TODAY関数）から正しくテンプレートを抽出できるか。
- **Detection Test**: 正常行、破損行（数値上書き）、異ロジック行が混在する中から破損箇所を特定できるか。
- **Repair Test**: 既存の値を破壊せず、数式のみを正確に修復できるか。

## 6. 安全策
- 修復前に `.bak` ファイルを自動生成。
- 修復実行後に `scripts/check_wbs_integrity.py` を再走らせ、論理的な整合性が維持されているか再検証。
