# Design: 将来予測 (EAC/ETC) 計算エンジン

## 1. 目的と価値
本コンポーネントは、現状の進捗効率（CPI/SPI）を基にプロジェクトの「着地点」を科学的に予測する、EVM分析エンジンのアップグレードである。
単なる現状把握を超え、将来のリスクを数値化することで、AIアドバイザーが「具体的かつ回避的なアクション」を提案可能にすることを目的とする。

### 核心的メリット
1. **科学的予見**: 「このままだと何人日オーバーするか」を PMBOK の公式に基づき算出する。
2. **多角的シナリオ**: 楽観・現実・慎重の3つの計算モデルを提供し、不確実性への備えを支援する。
3. **教育的メンタリング**: 予測値の背景にある PM 理論を AI が解説することで、ユーザーの管理能力向上を促す。

## 2. アーキテクチャ

### 構成要素
- `scripts/analyze_evm.py`: 将来予測ロジック（BAC, ETC, EAC, VAC）を追加実装。
- `EVMAnalyst.get_summary_json()`: AIアナリストが解釈しやすい構造化された予測データを追加。

### データフロー
1. **Load & Verify**: 既存の `WBSIntegrityChecker` でデータの健全性を担保。
2. **Current Metrics**: 現時点の PV, EV, AC, CPI, SPI を算出（既存）。
3. **Budget Baseline**: 全タスクの「工数予定」を合算し、**BAC (完成時総予算)** を特定。
4. **Forecasting**: BAC と現在の効率を基に、複数のシナリオで **EAC (完成時総コスト見積り)** を算出。
5. **AI Synthesis**: 計算結果を JSON に集約し、AIアナリストの推論エンジンへ渡す。

## 3. 計算ロジック (PMBOK準拠)

| 指標 | 名称 | 計算式 | 意味 |
| :--- | :--- | :--- | :--- |
| **BAC** | 完成時総予算 | `Σ(全行の工数予定)` | プロジェクト全体の総予算ベースライン。 |
| **ETC** | 残作業見積り | `(BAC - EV) / CPI` | 現時点から完了までにあとどれだけの工数が必要か。 |
| **EAC** | 完成時総コスト見積り | `AC + ETC` | 最終的にプロジェクトが完了した時の総コスト。 |
| **VAC** | 完成時差異 | `BAC - EAC` | 最終的な予算超過（または節約）の予測額。 |

### 予測シナリオ (Scenarios)
- **現実的 (Realistic)**: `EAC = BAC / CPI`
  - 現在の効率（CPI）が今後も継続すると仮定する標準的なモデル。
- **楽観的 (Optimistic)**: `EAC = AC + (BAC - EV)`
  - これまでの乖離は特殊要因であり、残りは計画通り（効率1.0）に進むと仮定。
- **慎重 (Pessimistic)**: `EAC = AC + [(BAC - EV) / (CPI * SPI)]`
  - コスト効率だけでなく、スケジュールの遅れも将来のコストに影響すると仮定。

## 4. インターフェース (AIアナリスト用JSON)

`get_summary_json()` に以下の `forecasting` セクションを追加する。

```json
"forecasting": {
    "bac": 150.0,
    "scenarios": {
        "realistic": {
            "eac": 165.0,
            "etc": 65.0,
            "vac": -15.0,
            "description": "現在の効率が継続した場合の予測"
        },
        "optimistic": { "eac": 155.0, "etc": 55.0, "vac": -5.0, "description": "残作業を計画通りに遂行した場合" },
        "pessimistic": { "eac": 180.0, "vac": -30.0, "description": "現在の効率と遅延が相互に影響した場合" }
    }
}
```

## 5. 検証計画 (DoD)
- [ ] `BAC` が基準日に左右されず、WBS全体の総予定工数を正しく合計していること。
- [ ] `CPI = 1.0` の時、`EAC = BAC` となること。
- [ ] `CPI < 1.0` (コスト超過) の時、`EAC > BAC` かつ `VAC < 0` となること。
- [ ] 3つのシナリオ（楽観・現実・慎重）が PMBOK 公式通りに算出されていること。
- [ ] AIアナリスト向けの JSON サマリーに `forecasting` セクションが含まれていること。

## 6. 規律: Script-as-Master
- Excel上のセルへの EAC/ETC 書き込みは、数式汚染を防ぐため、今回は最小限（または JSON 出力のみ）に留める。
- すべての予測は Python エンジンが「唯一の真実」として計算し、AI はその数値を基にアドバイスを行う。
