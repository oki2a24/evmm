# WBS/EVM マスターテンプレート デザインドキュメント

## 1. 目的
ユーザーが提供した `wbs_ebm_format.xlsx` の構造をベースに、自動計算ロジック、初学者向けガイド、および視覚的な分析機能を備えた「最強のプロジェクト管理テンプレート」を定義する。

## 2. アーキテクチャ
- **インターフェース**: Excel (.xlsx)
- **生成・操作**: Python (openpyxl / pandas)
- **構造**:
  - `Settings`: メンバーリスト、祝日、プロジェクト基本情報（開始日等）
  - `WBS_EVM`: メインの進捗管理と自動計算
  - `Dashboard`: SPI/CPI 推移グラフ、リソース負荷状況（自動集計）

## 3. 主要機能の詳細

### 3.1. 自動計算ロジック (EVM)
各タスクの 3 フェーズ（作成、レビュー、修正）に対して以下の計算を埋め込む。
- **PV (Planned Value / 計画値)**:
  - `IF(今日 < 開始予定, 0, IF(今日 > 終了予定, 工数予定, 工数予定 * (今日 - 開始予定) / (終了予定 - 開始予定)))`
- **EV (Earned Value / 出来高)**:
  - `工数予定 * 進捗率`
- **AC (Actual Cost / 実績コスト)**:
  - 直接入力された「工数実績」を参照。

### 3.2. 初学者向け UI/UX
- **用語の日本語併記**: 全ての EVM 指標に日本語訳を添える（例: `CPI (コスト効率指数)`）。
- **セルコメント**: 指標の列ヘッダーに、計算式の意味と「1.0を超えると良好、下回ると注意」といった解釈ガイドをコメントとして埋め込む。
- **条件付き書式**: CPI/SPI が 0.9 未満なら背景を赤、1.1 以上なら青にするなど、直感的なアラートを実装。

### 3.3. フェーズ構造の継承
提供されたフォーマットの「作成」「レビュー実施」「レビュー後修正」の 3 段階構造を維持し、それぞれの段階で個別に EVM を管理できるようにする。

## 4. データバリデーション
- 日付入力の妥当性チェック（終了日が開始日より前になっていないか）。
- 数式保護（誤って計算式を消さないよう、入力エリア以外をロックする設定を検討）。
