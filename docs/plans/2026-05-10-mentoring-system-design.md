# Design Document: 自己進化型メンタリング・システム

## 1. 目的 (Objective)
本システムは、EVM (Earned Value Management) の数値分析を基盤としつつ、プロジェクト固有の背景情報（コンテキスト）を学習・活用することで、ユーザーのプロジェクトマネジメント能力を養う「伴走型メンター」を提供することを目的とする。

## 2. コア・コンセプト (Core Concepts)

### 2.1 師匠モード (Socratic Mentoring)
- AIは答えを即答せず、数値の裏にある理論的意味をユーザーに問いかけ、思考を促す。
- PMBOK 理論に基づいた状況解釈のトレーニングを行う。

### 2.2 軍師モード (Context-Aware Advisor)
- プロジェクト固有の背景（人間関係、技術的課題、重要度）を考慮し、現場で使える「本質的な問いかけ」をユーザーと共に作り上げる。
- ユーザーからのフィードバックに基づき、アドバイスの精度を動的に向上させる。

### 2.3 自己進化型知識ベース (Self-Evolving Knowledge Base)
- 会話から得られた知見を `projects/*/docs/context.md` に蓄積し、セッションを跨いで記憶を保持する。

## 3. アーキテクチャ (Architecture)

### 3.1 ディレクトリ構造
```text
projects/
└── [project_name]/
    ├── *.xlsx         (実データ: EVM/WBS)
    └── docs/
        └── context.md  (背景知識: AIが読み書きする SSOT)
```

### 3.2 データフロー
1. **Load**: `analyze_evm.py` が Excel を解析し JSON を出力。同時に AI が `context.md` を読み込む。
2. **Reason**: AI が数値とコンテキストを照らし合わせ、不整合やリスクの兆候を特定。
3. **Dialogue**: 
   - AI がユーザーに問いかけ（師匠モード）。
   - ユーザーが現場の状況を回答。
   - AI がアドバイスを研磨（軍師モード）。
4. **Persist**: 合意された洞察を `context.md` に追記・更新。

## 4. `context.md` の仕様
AI が解析・更新しやすい Markdown 構造を維持する。
- **PJプロファイル**: 全体的な制約や特性。
- **リソース状況**: メンバーのスキルや役割。
- **タスク別洞察**: 特定のタスクに関連する定性的な背景。

## 5. 成功基準 (Success Criteria)
- AI が数値だけでなく、「過去にユーザーが話した背景」を引用して問いかけを行えること。
- 対話を通じて `context.md` が自動的に更新・整理されること。
- ユーザーが「現場で何を、誰に、なぜ確認すべきか」を論理的に説明できるようになること。
