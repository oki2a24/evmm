# Design: EVM 自動分析エンジン (EVMAnalyst)

## 1. 目的と価値
本コンポーネントは、WBSデータから正確なEVM指標（PV, EV, AC, CPI, SPI）を算出し、プロジェクトの現状診断と将来予測を行う「AIアナリスト」の心臓部である。

### 核心的メリット
1. **正確な計算**: Pythonによる厳密な数値計算により、AI(LLM)の計算ミスを排除する。
2. **過去の再現性**: 基準日（Status Date）を指定することで、過去の任意の時点での分析を可能にする。
3. **柔軟な深掘り**: 個人別、機能別、フェーズ別など、多角的なドリルダウン分析の土台を提供する。

## 2. アーキテクチャ

### 構成要素
- `scripts/analyze_evm.py`: 分析実行メインスクリプト。
- `scripts/utils.py`: 稼働日計算などの共通ユーティリティ（既存）。
- `EVMAnalyst` クラス: データのロード、EVM計算、レポート生成を担当。

### データフロー
1. **Load**: `pandas` を用いて WBS エクセルを読み込む。
2. **Pre-check**: `WBSIntegrityChecker` を呼び出し、データの論理的整合性を確認する（規律）。
3. **Calculate**: 
   - 指定された基準日に基づき、各行の PV, EV, AC を算出。
   - 全体、個人別、フェーズ別の集計（Roll-up）を実行。
4. **Predict**: CPI, SPI に基づき、完成予定日（EAC_t）と完成時総コスト（EAC）を算出。
5. **Output**: 
   - Excel の該当列（PV/EV/AC等）を `openpyxl` で外科的に更新。
   - ターミナルに PMBOK 理論に基づいた診断レポートを出力。

## 3. 計算ロジック (PMBOK準拠)

- **PV (Planned Value)**: 
  - `線形按分法`: `(基準日 - 開始予定日) / (終了予定日 - 開始予定日)` ※稼働日ベース
- **EV (Earned Value)**: 
  - `実績ベース`: `工数予定 × 進捗率(%)`
- **AC (Actual Cost)**: 
  - `直接入力ベース`: `工数実績` カラムの値を合算。
- **EAC (Estimate At Completion / 完成時総コスト予測)**:
  - `EAC = BAC / CPI` (現在の効率が継続すると仮定)
- **EAC_t (Schedule Prediction / 完成予定日予測)**:
  - `残稼働日数 / SPI` を現在の基準日に加算。

## 4. インターフェース

### コマンドライン
```bash
python scripts/analyze_evm.py <excel_path> --date 2026-04-03
```

### AIレポート構成
- **Summary**: CPI, SPI の値と、現在の「健康状態（信号機）」
- **Forecast**: 予定日からの乖離日数、予測コスト。
- **Drill-down**: 効率の低い（ボトルネックとなっている）個人または機能の特定。
- **Advice**: PMBOK理論に基づく具体的アクション（クラッシング、ファスト・トラッキングの検討など）。
