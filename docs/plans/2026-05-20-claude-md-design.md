# 設計ドキュメント: CLAUDE.md (プロジェクト憲法)

## 1. 概要
本ドキュメントは、Claude (AI) がプロジェクトにおいて遵守すべき指示書「`CLAUDE.md`」の設計案をまとめたものである。プロジェクトの核心である WBS/EVM エクセル・エコシステムの整合性を、人間の編集を許容しつつ、いかにしてプログラムによって担保するかを定義する。

## 2. 設計思想: 「検証と修復 (Validation & Recovery)」
「Excel の直接編集は現実的な運用である」という前提を受け入れる。ただし、そのリスク（数式の破壊、参照エラー）を、プロジェクトに組み込まれた「検証スクリプト」と「修復ロジック」によって最小化する。

## 3. 動作原理
- **許容**: ユーザーによる自由な Excel 編集を妨げない。
- **検知**: 編集後のファイルに対して、Claude が検証スクリプトを実行し、整合性をチェックする。
- **修復**: 検知されたエラーに対し、Claude がスクリプトやマスターデータに基づいた修復案を提示、または実行を試みる。

## 4. 動作指針 (Claude の行動規範)
1. **変更検知の徹底**: ファイル変更（`git diff` 等）を検知した際、またはユーザーから変更の報告を受けた際、直ちに検証スクリプトを実行する。
2. **検証の実行**: `python scripts/check_integrity.py` (仮) などのスクリプトを動かし、構造的な破損がないか確認する。
3. **エラー報告と修復提案**: 検知されたエラーの内容を具体的に報告し、修復プロセス（テンプレートへの再適用、マスターデータによる上書き等）をユーザーに提案、または許可を得て実行する。

## 5. 期待される効果
- データの整合性維持
- 人的ミスによるプロジェクト全体の破壊（テンプレート破損）の防止
- 復旧コストの低減
